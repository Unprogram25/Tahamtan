# Tahamtan — ابزار گرافیکی ارتباط سریال و شبکه
نسخه: 1.0.0  
نویسنده: Afshin Moradzadeh (Dev by Afshin Moradzadeh)  
محل ذخیرهٔ پیکربندی: `config.json` (در مسیر جاری که برنامه اجرا می‌شود)

توضیحات
--------
Tahamtan یک برنامهٔ گرافیکی کوچک و کاربردی برای برقراری ارتباط با دستگاه‌ها از طریق پورت سریال (Serial) و پروتکل TCP است. این ابزار برای پیکربندی، تست، مانیتورینگ و عیب‌یابی دستگاه‌های امبدد، میکروکنترلرها و تجهیزات شبکه مفید است.

ویژگی‌ها
--------
- تب جداگانه برای ارتباط سریال و TCP Client
- اسکن و انتخاب پورت‌های سریال محلی (USB/COM)
- انتخاب Baud Rateهای رایج و ذخیرهٔ آخرین انتخاب در `config.json`
- ارسال و دریافت داده با نمایش رنگی (sent/received/error/warning)
- چراغ وضعیت (Disconnected / Connecting / Connected)
- پشتیبانی از قالب انتهای خط (EOL): `\r\n`, `\n`, `\r`, یا None
- دستورات از پیش تعریف شده (Preset commands): `/lcd`, `/ping`, `/lan`, `/ttl`, `/rs-422`, `/usb`, `/conf`, `/reset`
- برای دستور `/conf` امکان انتخاب کلید پیکربندی و مقداردهی سریع با مقادیر پیش‌فرض
- قابلیت Ping (محاسبه RTT) در تب TCP
- صوت هشدار هنگام دریافت داده (با استفاده از winsound در ویندوز)
- میانبر صفحه‌کلید Ctrl+L برای پاک کردن پنجرهٔ وضعیت

ملزومات (Requirements)
-----------------------
- Python 3.8 یا بالاتر
- کتابخانه‌ها (قابل نصب با pip):
  - pyserial
  - ping3
- Tkinter (معمولاً همراه نصب Python است)
- winsound (کتابخانهٔ استاندارد ویندوز؛ برای صدا در ویندوز استفاده می‌شود)

نصب وابستگی‌ها
---------------
پیشنهاد می‌شود در یک virtual environment کار کنید. سپس:
```
pip install pyserial ping3
```

اجرای برنامه
------------
از مسیر ریشهٔ پروژه:
```
python Tahamtan.py
```

نحوهٔ استفاده
--------------
1. اجرای برنامه و انتخاب تب موردنظر (Serial یا TCP Client).
2. Serial:
   - با زدن دکمهٔ Refresh، پورت‌های سریال شناسایی می‌شوند.
   - پورت و Baud Rate را انتخاب کرده و روی Connect کلیک کنید.
   - پس از اتصال، می‌توانید پیام وارد کنید یا از Presetها استفاده کنید و Send بزنید.
   - اگر گزینهٔ Show \r\n فعال باشد، کاراکترهای EOL در پنجرهٔ لاگ نمایش داده می‌شوند.
   - دستور `/reset` پس از ارسال، دستگاه را ریست می‌کند و ارتباط بسته می‌شود (مطابق کد).
3. TCP Client:
   - IP و Port را وارد کنید و Connect را فشار دهید.
   - با موفقیت اتصال، می‌توانید پیام ارسال و دریافت کنید.
   - دکمهٔ Ping RTT را محاسبه و نمایش می‌دهد (در صورت نیاز به سطح دسترسی برای ping، خطا نمایش داده می‌شود).

فرمت ارسال پیام و EOL
---------------------
- در بخش ارسال (Send) می‌توانید EOL مناسب را انتخاب کنید:
  - `\r\n` (پیش‌فرض برای Serial)
  - `\n`
  - `\r`
  - `None` (بدون EOL)
- با گزینهٔ "Show \r\n" می‌توانید نمایش متن ارسالی را به‌گونه‌ای ببینید که کاراکترهای السیری بصورت `\r` و `\n` نشان داده شوند.

دستورات از پیش تعریف شده (Preset Commands)
-----------------------------------------
- لیست اولیه: `["", "/lcd ", "/ping ", "/lan ", "/ttl ", "/rs-422 ", "/usb ", "/conf ", "/reset"]`
- اگر `/conf ` را انتخاب کنید، یک Combobox برای انتخاب کلید پیکربندی نمایش داده می‌شود و مقدار پیش‌فرض آن در سمت راست قرار می‌گیرد.
- کلیدها و مقادیر پیش‌فرض پیکربندی:
  - ttl: 1
  - rs-422: 1
  - usb: 1
  - lan: 1
  - alcd: 1
  - server-ip: 192.168.1.200
  - server-port: 8585
  - rs-422-baudrate: 115200
  - rs-422-frame: 8n1
  - ttl-baudrate: 115200
  - ttl-frame: 8n1
  - alcd-rows: 2
  - alcd-columns: 16

ذخیرهٔ پیکربندی
----------------
- بعد از اتصال سریال، پورت و Baud Rate در `config.json` ذخیره می‌شود و در اجرای بعدی برنامه تلاش می‌شود آن مقادیر بارگذاری شوند.

نکات و محدودیت‌ها
-----------------
- قابلیت پخش صدا با winsound فقط در ویندوز در دسترس است. در لینوکس/مک برای صدا باید از راهکار جایگزین استفاده کنید.
- برای اجرای تابع Ping ممکن است نیاز به دسترسی‌های سطح پایین (raw socket) یا دسترسی ادمین/روت باشد؛ اگر خطای PermissionError مشاهده کردید، برنامه را با دسترسی مناسب اجرا کنید یا از روش دیگری برای بررسی reachability استفاده کنید.
- خواندن/نوشتن سریال و سوکت‌ها در تردهای جدا انجام می‌شود تا رابط کاربری مسدود نشود.
- پیام‌های دریافتی به صورت UTF-8 رمزگشایی می‌شوند؛ در صورت وجود کاراکترهای غیرقابل‌رمزگشایی ممکن است خطا بدهد.

عیب‌یابی (Troubleshooting)
--------------------------
- خطای "Could not open the port": اطمینان حاصل کنید پورت انتخابی آزاد است و برنامهٔ دیگری آن را نگرفته است.
- اگر پینگ کار نمی‌کند: بررسی کنید IP درست است، فایروال یا محدودیت سیستمی وجود نداشته باشد و در صورت نیاز برنامه را با سطوح دسترسی بیشتر اجرا کنید.
- اگر پیام‌های دریافتی نمایش داده نمی‌شوند: مطمئن شوید Baud Rate و پارامترهای فریمینگ درست تنظیم شده‌اند و timeout پورت مناسب است.
